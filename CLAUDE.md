# Word Cross Quiz 프로젝트 규칙

## 프로젝트 개요

교사가 전자칠판에 띄우면 학생이 QR 코드를 스캔해 스마트폰·태블릿에서 푸는 한국어 낱말 퀴즈 웹 앱.  
각 퀴즈는 독립된 폴더로 관리하며, 이미지를 받으면 Claude가 분석해 앱을 생성한다.

---

## 폴더 구조

```
wordcross/                     ← 루트 (git 저장소)
├── CLAUDE.md                  ← 프로젝트 공통 규칙
├── .gitignore
└── {과목폴더}/                ← 예: SportsC, MathA, ScienceB
    └── {단원폴더}/            ← 예: 1단원, 2단원
        ├── index.html
        └── {단원명}_hint.md
```

### 폴더 네이밍 규칙

- **과목 폴더**: `{과목명}` (영문, 대소문자 혼용 허용) — 예: `SportsC`, `MathA`
- **단원 폴더**: `{N}단원` — 예: `1단원`, `2단원`

---

## 기술 스택

- **Vanilla HTML / CSS / JS** — 빌드 툴 없음, `index.html` 단일 파일로 완결
- 외부 라이브러리는 CDN으로 로드 (오프라인 불필요 시)
- QR 코드 생성: `qrcode.js` CDN 사용 (클라이언트 사이드 생성)

---

## 반응형 레이아웃 규칙

학생의 스마트폰·태블릿에서 동작해야 하므로 아래를 반드시 준수한다.

- `<meta name="viewport" content="width=device-width, initial-scale=1.0">`
- 격자 셀 크기: `vmin` 단위 사용 — 화면 방향(가로·세로)에 관계없이 격자가 화면에 맞게 축소·확대
- 최소 터치 영역: 셀 한 변 최소 `44px` (모바일 터치 접근성 기준)
- **힌트 영역: 격자 위(상단)에 배치** — 스크롤 가능한 영역
- 전체 레이아웃: Flexbox 세로 배치 (`column`) — **힌트 → 격자 → 버튼** 순서

---

## QR 코드 규칙

- **표시 방식: 탭 UI** — 퀴즈 탭 / QR 코드 탭 두 개로 분리
  - 퀴즈 탭: 힌트 + 격자 + 버튼
  - QR 코드 탭: QR 코드만 크게 표시 (화면 세로의 약 70%, `70vh`)
- **QR 탭은 모바일에서 숨김** (`@media (max-width: 768px) { .tab-qr { display: none; } }`)  
  → 이미 접속한 학생 화면에는 QR 탭 불필요
- **URL**: 각 앱의 `index.html` 상단 `const APP_URL = "..."` 변수에 배포 URL을 하드코딩
  - 배포 전에는 `"YOUR_DEPLOY_URL_HERE"` 플레이스홀더로 남겨둔다
  - 배포 후 사용자가 이 값만 교체하면 QR이 자동 갱신됨
- **가시성**: QR 주변에 흰 여백(quiet zone) 충분히 확보, 배경 흰색
- **안내 문구**: QR 아래 `"QR 코드를 스캔하여 접속하세요"` 텍스트 표시 (칠판에서 읽힐 크기)

---

## 퀴즈 데이터 구조

각 앱의 JS 내부에 아래 형식으로 하드코딩한다.

```js
const PUZZLE = {
  cols: 10,       // 격자 가로 셀 수
  rows: 10,       // 격자 세로 셀 수
  cells: [
    // [row, col, 정답글자, 방향힌트번호(가로/세로)]
    // 빈 칸(검정)은 데이터에 포함하지 않음
  ],
  words: [
    {
      id: 1,
      direction: "across",   // "across" | "down"
      row: 0, col: 0,        // 시작 좌표 (0-indexed)
      answer: "축구",
      hint: "발로 공을 차는 스포츠"
    },
    // ...
  ]
};
```

## 이미지 분석 → 코딩 절차

사용자가 퀴즈판 이미지를 제공하면 아래 순서로 진행한다.

1. 이미지에서 격자 크기(가로×세로 셀 수) 파악
2. 검정 칸(블로킹 셀) 위치 파악
3. 각 단어의 시작 좌표, 방향, 정답 파악
4. 힌트 번호 매핑 (가로 → 세로 순, 좌→우, 위→아래)
5. 위 `PUZZLE` 구조로 데이터화 후 `index.html` 생성
6. 분석 결과를 먼저 텍스트로 사용자에게 확인 요청 후 코딩

## UI/UX 패턴

### 셀 상태 색상
| 상태 | 색상 |
|------|------|
| 기본 | 흰색 |
| 선택됨 | 하늘색 (`#90CAF9`) |
| 정답 | 연초록 (`#A5D6A7`) |
| 오답 | 연빨강 (`#EF9A9A`) |
| 블로킹(검정) | 검정 (`#212121`) |

### 한국어 입력 처리
- 셀 클릭 시 `<input>` 포커스, `maxlength="1"`, `inputmode="text"`
- IME 조합 중 (`compositionstart` ~ `compositionend`) 에는 값 확정 안 함
- `compositionend` 이벤트에서 첫 글자만 추출해 셀에 기록

### 정답 확인 버튼
- 버튼 클릭 시 모든 입력값을 정답과 비교
- 맞은 셀 → 연초록으로 표시
- **틀리거나 빈 셀 → 연빨강 배경 + 정답 글자를 빨간색으로 표시** (입력값 대체)
- 전체 정답 시 축하 메시지 표시
