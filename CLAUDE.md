# Word Cross Quiz — 프로젝트 공통 규칙

## 프로젝트 개요

교사가 전자칠판에 띄우면 학생이 QR 코드를 스캔해 스마트폰·태블릿에서 푸는 한국어 낱말 퀴즈 웹 앱.  
각 퀴즈는 독립된 폴더로 관리하며, 이미지를 받으면 Claude가 분석해 앱을 생성한다.

---

## 폴더 구조

```
wordcross/                     ← 루트 (git 저장소)
├── CLAUDE.md                  ← 프로젝트 공통 규칙
├── .gitignore
└── {과목폴더}/                ← 예: sports_C, mathA
    ├── CLAUDE.md              ← 퀴즈 유형별 규칙
    └── {단원폴더}/            ← 예: chap1, chap2
        ├── index.html
        └── {단원명}_hint.md
```

### 폴더 네이밍 규칙

- **과목 폴더**: `{과목명}` (영문, 소문자+언더스코어 허용) — 예: `sports_C`, `math_A`
- **단원 폴더**: `chap{N}` — 예: `chap1`, `chap2`

---

## 기술 스택

- **Vanilla HTML / CSS / JS** — 빌드 툴 없음, `index.html` 단일 파일로 완결
- 외부 라이브러리는 CDN으로 로드
- QR 코드 생성: `qrcode.js` CDN 사용 (클라이언트 사이드)

---

## 반응형 레이아웃 규칙

학생의 스마트폰·태블릿에서 동작해야 하므로 아래를 반드시 준수한다.

- `<meta name="viewport" content="width=device-width, initial-scale=1.0">`
- 격자 셀 크기: `vmin` 단위 사용 — 화면 방향에 관계없이 격자가 화면에 맞게 축소·확대
- 최소 터치 영역: 셀 한 변 최소 `44px` (모바일 터치 접근성 기준)
- 전체 레이아웃: Flexbox 세로 배치 (`column`) — **힌트 → 격자 → 버튼** 순서

---

## QR 코드 규칙

- **표시 방식: 탭 UI** — 퀴즈 탭 / QR 코드 탭 두 개로 분리
  - QR 코드 탭: QR 코드만 크게 표시 (화면 세로의 약 70%, `70vh`)
- **QR 탭은 모바일에서 숨김** (`@media (max-width: 768px) { .tab-qr { display: none; } }`)
- **URL**: 각 앱의 `index.html` 상단 `const APP_URL = "..."` 변수에 배포 URL 하드코딩
  - 배포 전에는 `"YOUR_DEPLOY_URL_HERE"` 플레이스홀더로 남겨둔다
- **안내 문구**: QR 아래 `"QR 코드를 스캔하여 접속하세요"` 텍스트 표시

---

## 각 앱에 반드시 포함할 요소

- [ ] 퀴즈 제목 (`<h1>`)
- [ ] QR 코드 섹션 (PC/칠판에서만 표시)
- [ ] 반응형 격자
- [ ] 정답 확인 버튼
- [ ] 다시 풀기 버튼 (입력 초기화)
- [ ] `APP_URL` 변수 (배포 URL 플레이스홀더)
